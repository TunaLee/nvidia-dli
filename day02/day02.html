CH2융합

문제 — 두 점수는 못 더한다

🥇

서로 다른 시험의 점수

국어 원점수와 수학 원점수는 못 더한다 — 대신 ‘석차(등수)’로 합치면 공정하다. 그게 RRF의 발상.

/
CH2RRF

RRF — 점수가 아니라 ‘순위’로 합친다

RECIPROCAL RANK FUSION
score(d) = Σ 1 / (k + 순위i(d))
k=60 (대개 그대로)
두 목록에서 ‘위에’ 오를수록 높은 점수
✓
순위만 보므로 척도 문제 없음
✓
양쪽에서 고르게 높은 문서가 이긴다
✓
단순·강건 → 실무 기본 융합
/
CH2미니 실습 · 손계산

직접 해보기 — RRF로 1등 뽑기

과제 — k=60, score=1/(k+순위)

D1
벡터 1위, BM25 3위 → 1/61 + 1/63 = ?
D2
벡터 2위, BM25 1위 → 1/62 + 1/61 = ?

정답

D1≈0.0323, D2≈0.0325 → D2가 근소 우위. ‘양쪽에서 고른’ 문서가 이긴다.

CH3문제

‘유사도 1등’이 진짜 1등이 아닐 때

!

→
관련 있는 후보는 모았으니, 이제 정밀하게 다시 줄세우자
아이디어
넓게 줍고(1차) → 꼼꼼히 고른다(2차). 그 2차가 reranker.
/
CH3구조 차이

따로 보기 vs 나란히 보기

Bi-encoder(1차)

질문·문서를 따로 벡터화해 비교. 빠르지만 미묘함을 놓침.

Cross-encoder(reranker)

질문+문서를 함께 읽고 관련성 채점. 정확하지만 느림.

🪪

사진 매칭 vs 대면 비교

bi=두 사람 사진을 각자 보고 닮았나 추측 / cross=둘을 나란히 세워 직접 비교. 당연히 후자가 정확하다.

/
CH3핵심 비유

Reranker = ‘면접’ 단계

1차
서류전형 — 많이·빠르게 추린다(검색, N개)
2차
면접 — 소수를 꼼꼼히 다시 평가(reranker, K개)
🧑‍💼

왜 둘 다

모두를 면접할 순 없다(느림). 서류로 좁힌 뒤 면접한다 — 그래서 ‘2단계’.

/
CH3파이프라인

넓게 줍고(N) → 정밀하게 고른다(K)

①
하이브리드
후보 N≈50
②
Reranker
관련성 재채점
③
Top-K
최종 K≈5
④
생성
근거로 답

N과 K

N 너무 작으면 면접할 후보가 없고, 너무 크면 느리다. N≈30~50 → K≈3~6이 흔한 출발점.

/
CH3미니 실습 · 판단

직접 해보기 — 면접에 누굴 올릴까

과제 — 1차 검색이 후보 50개를 줬다. 다음 중 reranker로 넘길 N은?

A
50개 전부를 LLM 컨텍스트에 그냥 넣는다
B
상위 50을 reranker로 채점 → 상위 5만 생성에
C
1개만 검색해서 바로 답한다

정답

B. A는 잡음·비용↑(가운데가 잊힘), C는 근거 부족. 넓게 줍고 → 좁혀 면접이 정석.

import re
from rank_bm25 import BM25Okapi
tok = lambda t: re.findall(r"[A-Za-z0-9\-]+|[가-힣]+", t)  # 조사 분리
tokenized = [tok(t) for t in texts]
bm25 = BM25Okapi(tokenized)
def bm25_search(q, k=10):
    s = bm25.get_scores(tok(q))
    return sorted(range(len(s)), key=lambda i:-s[i])[:k]

벡터+BM25를 RRF로 합치기

lab2_hybrid.py
def rrf(rank_lists, k=60):
    score={}
    for ranks in rank_lists:
        for r,idx in enumerate(ranks):
            score[idx]=score.get(idx,0)+1/(k+r+1)
    return sorted(score, key=lambda i:-score[i])
def hybrid(q,n=50):
    return rrf([vector_search_idx(q,n), bm25_search(q,n)])[:n]
✅ 벡터만 / BM25만 / 하이브리드 결과를 같은 질문으로 비교

CH9LAB 3 · 재정렬(면접)

Cross-Encoder로 Top-K 재정렬

lab3_rerank.py
from sentence_transformers import CrossEncoder
rk = CrossEncoder("cross-encoder/mmarco-mMiniLMv2-L12-H384-v1")  # 로컬·다국어
def rerank(q, cand, top_k=5):
    pairs=[(q, texts[i]) for i in cand]
    sc=rk.predict(pairs)
    order=sorted(range(len(cand)), key=lambda j:-sc[j])
    return [cand[j] for j in order[:top_k]]
best = rerank(q, hybrid(q,n=50), top_k=5)   # 서류50→면접→5

def multi_query(q, n=30):
    p=f"질문을 뜻은 같되 표현이 다른 3개로, 한 줄씩:\n{q}"
    vs=client.chat.completions.create(model=LLM_MODEL,
        messages=[{"role":"user","content":p}]
        ).choices[0].message.content.splitlines()
    return rrf([vector_search_idx(v,n) for v in vs if v.strip()])[:n]

lab7_graphrag.py
import networkx as nx
# 1) LLM(Qwen)으로 문서에서 (주체, 관계, 대상) 트리플 추출 → JSON
triples = extract_triples(docs)          # [("X-200","제조사","ACME"), ...]
# 2) 지식 그래프 구성
G = nx.DiGraph()
for s_, r, o in triples:
    G.add_edge(s_, o, rel=r)
