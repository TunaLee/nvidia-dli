# 하네스 선택 가이드

하네스 도구는 종류가 많다. 다 써볼 필요는 없다. **내 실패가 어떤 모양인지** 먼저 정하고, 거기 맞는 것만 얹는다.

## 증상 → 처방

| 증상 | 처방 | 무엇을 하는가 | 강의 |
|---|---|---|---|
| 매 세션 컨벤션을 다시 설명한다 | **CLAUDE.md** | 룰을 한 번만 적재한다. 프롬프트 캐싱이 붙어 비용도 내려간다 | Day08 |
| 쉬운 일에 비싼 모델을 쓴다 | **모델 라우팅** | 단순은 Haiku, 구현은 Sonnet, 설계는 Opus. 품질은 그대로 두고 비용만 내린다 | Day08 |
| 출력이 장황하다 · 토큰이 샌다 | **입력 줄이기** → caveman · ponytail | 절감의 대부분은 입력에서 온다. 출력 압축은 그다음이다 | Day08 |
| 위험한 명령을 그냥 실행한다 | **Hook** (`guard.mjs`) | 도구 호출 직전에 코드로 막는다. exit 2면 진짜 차단이다 | Day09 |
| 요청 안 한 기능을 자꾸 덧붙인다 | **gstack** | 역할 페르소나로 제품 리뷰를 먼저 태워 범위를 깎는다 | Day09 |
| 구현이 자꾸 깨진다 · 테스트를 건너뛴다 | **Superpowers** | 실패 테스트 없이는 코드를 못 쓰게 한다. 단계 건너뛰기를 차단한다 | Day09 |
| 큰 작업을 한 세션에 다 시킨다 | **오케스트레이션** | 분해 → 위임 → 병렬 → 검증. 리드는 위임만 한다 | Day09 |

증상이 여러 개면 하나씩 처방한다. 한 번에 다 얹으면 무엇이 효과를 냈는지 알 수 없다.

## 강제냐 부탁이냐

이게 가장 중요한 구분이다.

| 수단 | 성격 | 무시될 수 있나 |
|---|---|---|
| **CLAUDE.md** | 컨텍스트 — 따르려고 읽는 문서 | **그렇다** |
| **tools 제한** | 권한 — 애초에 할 수 없다 | 아니다 |
| **Hook** | 강제 — 도구 호출 직전에 차단 | 아니다 |

CLAUDE.md에 "운영 DB를 지우지 마라"고 적어두는 것과, 그 명령을 hook으로 막는 것은 전혀 다른 일이다. Replit 사고는 전자로는 막히지 않는다는 걸 보여줬다.

**CLAUDE.md는 부탁, Hook은 강제.** 안전은 부탁으로 만들어지지 않는다.

## Superpowers와 gstack

| | Superpowers | gstack |
|---|---|---|
| 성격 | 방법론을 코드로 강제한다 (Hard Gate) | 가상 개발 팀 — 역할 페르소나 |
| 명령 수 | 적다 | 많다 |
| 방식 | 강제 흐름 — brainstorm → plan → execute, 건너뛰기 차단 | 골라 쓰기 — `/office-hours` · `/plan-*-review` · `/qa` · `/ship` |
| 호환 | Claude Code · Codex · Cursor 등 여러 도구 | 주로 Claude Code |
| 언제 | 품질이 무너질 때. 테스트를 강제해 되돌린다 | 제품 관점 검토가 필요할 때 |

명령 수 자체가 두 도구의 성격이다. 한쪽만 골라야 하는 것도 아니다 — 강점이 달라 서로 보완한다.

## 규모에 맞게

| 규모 | 도구 조합 |
|---|---|
| 소 — 한 시간 이내 | 직접 한다. 도구 없음 |
| 중 — 반나절 | gstack |
| 대 — 며칠 | gstack + Superpowers |
| 특대 — 일주일 이상 | gstack + Superpowers로 마일스톤을 나눠 길게 끌고 간다 |

작은 일에 큰 하네스를 씌우면 오버헤드만 남는다.

## 멀티에이전트로 갈지 — 판단 3문

1. **병렬로 분해할 수 있는가?**
2. **그 가치가 약 15배 토큰보다 큰가?**
3. **사람이 검수할 수 있는 단위인가?**

**하나라도 아니면 단일 에이전트 + Plan Mode.** 멀티는 기본값이 아니다.

## 설치 전 주의

**1. 한 저장소에 다 깔지 않는다.** 여러 프레임워크를 같은 저장소에 설치하면 `.claude/` 안의 슬래시 커맨드·에이전트·스킬 이름이 겹친다. 어느 쪽 정의가 이겼는지 모르는 채로 돌게 된다. 폴더를 나눠 하나씩 시험한다.

**2. 광고 수치를 그대로 믿지 않는다.** 출력 압축 도구의 "65퍼센트 절감" 같은 문구는 출력 토큰만의 이야기다. 입력(컨텍스트·파일·도구 결과)이 비용의 대부분을 차지하는 경우가 많다. 수치는 직접 재본다. 재보는 것 자체가 실습이다.

**3. 요금제 한도를 미리 확인한다.** Superpowers는 task마다 서브에이전트를 띄운다. 한도에 걸리면 작업 도중에 멈춘다.
