# 하네스 선택 가이드

하네스 프레임워크는 종류가 많다. 다 써볼 필요는 없다. **내 실패가 어떤 모양인지** 먼저 정하고, 거기 맞는 것만 얹는다.

## 증상 → 처방

| 증상 | 처방 | 무엇을 하는가 | 강의 |
|---|---|---|---|
| 요청 안 한 기능을 자꾸 덧붙인다 | **GSTACK** | 23개 역할(PM·엔지니어·QA·보안…) 페르소나. 제품 리뷰를 먼저 태워 범위를 깎는다 | Day08 |
| 출력이 장황하다 · 토큰이 샌다 | **ponytail** · **caveman** | ponytail은 YAGNI로 꼭 필요한 코드만 짜게 한다(적게 짜기). caveman은 출력을 압축한다(적게 말하기) | Day08 |
| 어제보다 품질이 떨어진다 | **Superpowers** | 실패 테스트 없이는 코드를 못 쓰게 한다. task마다 새 서브에이전트 + 2단계 리뷰 | Day09 |
| 한 시간 지나면 헛소리를 한다 | **GSD** | 스테이지마다 얇은 오케스트레이터가 전문 서브에이전트를 새로 스폰한다. 각자 깨끗한 컨텍스트 | Day09 |
| 내 도메인 전용 팀이 필요하다 | **revfactory/harness** | 도메인 한 줄을 넣으면 에이전트 팀과 스킬을 찍어내는 메타 팩토리 | Day08에서 생성 → Day09에서 실행 |

증상이 여러 개면 하나씩 처방한다. 한 번에 다 얹으면 무엇이 효과를 냈는지 알 수 없다.

## 단일과 멀티를 가르는 기준

기준은 하나다 — **서브에이전트를 새로 스폰하는가.**

| | 스폰 | 구조 | 강의 |
|---|---|---|---|
| GSTACK | 안 한다 | 단일 세션에서 역할(페르소나)만 바꾼다 | Day08 |
| ponytail · caveman | 안 한다 | 스킬 하나로 출력 습관을 바꾼다 | Day08 |
| Superpowers | 한다 | task마다 새 서브에이전트, 리뷰는 2단계 | Day09 |
| GSD | 한다 | 스테이지마다 전문 서브에이전트를 새로 스폰 | Day09 |

GSTACK은 역할이 23개라 팀처럼 보이지만 세션은 하나다. 컨텍스트를 공유하고, 사람이 배역을 갈아입듯 프롬프트만 바뀐다. 그래서 Day08(단일 에이전트 하네스)이다.

Superpowers와 GSD는 새 세션을 띄운다. 서브에이전트는 자기 컨텍스트를 새로 채우고, 결과만 부모에게 돌려준다. 긴 작업에서 컨텍스트가 썩지 않는 이유가 이것이고, Day09(멀티에이전트)인 이유도 이것이다.

대신 **멀티는 토큰을 많이 쓴다.** 서브에이전트 하나가 곧 세션 하나다. 스테이지가 다섯이면 다섯 번 채운다. 강력한 만큼 비싸다는 것을 계산에 넣고 시작한다.

## 설치 전 주의

**1. 한 저장소에 다 깔지 않는다.** 여러 프레임워크를 같은 저장소에 설치하면 `.claude/` 안의 슬래시 커맨드·에이전트·스킬 이름이 겹친다. 어느 쪽 정의가 이겼는지 모르는 채로 돌게 된다. 폴더를 나눠 하나씩 시험한다.

**2. 광고 수치를 그대로 믿지 않는다.** caveman의 광고 문구는 "65퍼센트 절감"이지만 실측은 약 8.5퍼센트였다. 줄어드는 건 출력 토큰뿐이고, 입력(컨텍스트·파일·도구 결과)이 대부분을 차지하기 때문이다. 수치는 직접 재본다. 재보는 것 자체가 실습이다.

**3. 요금제 한도를 미리 확인한다.** Superpowers·GSD는 서브에이전트를 많이 스폰한다. 한도에 걸리면 작업 도중에 멈춘다.

## 설치

```powershell
# Superpowers (Day09)
/plugin marketplace add obra/superpowers
/plugin install superpowers

# GSD (Day09)
npx get-shit-done-cc --claude --local
npx get-shit-done-cc --codex        # 같은 프레임워크를 Codex에도
```
