# 파이널 프로젝트 — LLM 에이전트 호스트 웹앱

주제 자유. LLM 에이전트를 탑재한 웹 앱을 만들고 시연으로 평가.

## 필수 요건

**1. 웹 앱**

- 프레임워크 자유 (수업 기준: Next.js)
- 채팅 또는 그에 준하는 상호작용 UI
- LLM 응답 스트리밍 표시
- 로딩·에러 상태 처리 (API 실패 시 빈 화면 금지)

**2. LLM 에이전트**

- 판단 → 도구 → 관찰 루프 (7강 구조)
- 직접 정의한 도구 1개 이상 — LLM이 상황에 맞게 선택·호출
- 모델: NVIDIA API (Qwen) 기준, 다른 관리형·자가 서빙 허용
- API 키는 환경변수로 (하드코딩 금지)

## 가점 항목

시연에서 실제로 동작해야 인정. 개수보다 완성도 우선.

- MCP 서버 연동 · 멀티에이전트 오케스트레이션 · DB 저장 · 소셜 로그인 · 웹검색 도구 · 배포 · 하이브리드 서빙

## 평가 기준

- 동작·완성도 · 에이전트 구현 · 현실성 · 창의성 · 디자인·UX
- 시연 후 아키텍처 문답 (요청 → 응답 경로를 본인이 설명)

## 진행 방식

1. **주제 피칭** — 시작 후 이른 시점에 주제와 핵심 기능을 강사에게 승인. 범위가 크면 이 단계에서 축소.
2. **개발** — 코드 에이전트(Claude Code · Codex) 사용 권장. 8~9강에서 만든 앱에서 출발하거나 새로 시작 모두 가능. 평가 대상은 코드가 아니라 동작하는 프로덕트와 본인의 이해.
3. **시연** — localhost 허용. 만일에 대비한 핵심 흐름 화면 녹화 백업 권장.

## 제출물

- 깃 저장소 (공개 또는 강사 초대)
- README: 주제 한 줄 · 간단한 아키텍처 그림 · 실행 방법 · 적용한 가점 항목
- `.env` 커밋 금지 — `.env.example`로 키 이름만 공유

## 주제 예시

그대로 만들어도 무방. 백지에서 고민하는 시간을 줄이는 용도.

- 여행 플래너 — 웹검색 기반 일정·명소 수집과 계획표 생성
- 가계부 비서 — 지출 내역 DB 축적과 자연어 조회·분석
- 회의록 정리기 — 녹취 텍스트에서 결정사항·할 일 추출
- 뉴스 리서처 — 주제 검색·요약·비교 보고서 작성
- 레시피 도우미 — 재료 기반 추천과 장보기 목록 생성
- 코드 리뷰 봇 — 저장소 파일 읽기와 리뷰 코멘트 생성

## 주의

- NVIDIA API rate limit — 시연 직전 리허설 호출 아끼기. 한도 문제 시 강사에게 백업 엔드포인트 요청.
- 시연 도중의 실패는 감점 요인. 단, 에러를 처리해 보여주는 친절한 실패는 완성도로 인정.
