CAPSTONE필수

이 네 가지는 꼭

1

여러 도구로 된 MCP 서버

최소 하나는 행동(부작용) Tool, 하나는 읽기/검색. Inspector로 확인. (3강)

2

친절한 에러·입력 검증

잘못된 입력에 ‘무엇을·어떻게 고쳐라’를 알려준다. (5강)

3

에이전트 데모

Qwen이 tool-calling으로 내 도구를 골라 질문을 푸는 노트북. (5강)

4

배포

Horizon(공개 URL) 또는 로컬 HTTP로 호출 가능하게. (5강)

/
CAPSTONE선택

주제에 맞으면 녹인다 (억지 X)

→
검색을 진짜 RAG로 — 임베딩·하이브리드·리랭커 (1~2강)
→
Resource + Prompt — 읽기 리소스, 재사용 프롬프트, 서버 합성 (4강)
→
똑똑한 기능 — 구조화 출력 · sampling · elicitation · Context · annotations (5강)
주의
다 넣지 말 것 — 주제에 자연스럽게 맞는 것만.

CAPSTONE미리보기

이런 모양이 된다

server.py
@mcp.tool
def add_note(text: str) -> dict:
    """메모를 저장한다"""
    NOTES.append(text)
    return {"저장됨": text}
agent.py
async with Client(mcp) as c:
    tools = to_openai_tools(await c.list_tools())
    # Qwen이 tool_calls 로 도구를 고른다
    out = await c.call_tool(name, args)
