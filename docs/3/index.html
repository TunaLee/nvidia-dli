CH0도입

왜 매번 ‘연결’이 문제인가

✕ 표준이 없을 때

모델 M개 × 도구 N개 = M×N개의 맞춤 연동을 각자 만든다. 도구가 늘 때마다 모든 앱이 다시 작업.

정의
Model Context Protocol — LLM 앱이 외부 데이터·도구를 표준화된 방식으로 안전하게 쓰게 하는 프로토콜. 2024년 11월 공개 → 사실상 표준으로 확산.
→

→

/
CH1구성요소

호스트 · 클라이언트 · 서버

Host
AI 앱(Claude Desktop 등)
Client
호스트 안의 연결 담당
Server
도구·데이터 제공(우리가 만든다)
역할
호스트가 LLM을 품고, 클라이언트가 서버와 1:1로 말하며, 서버가 실제 기능을 노출한다. 오늘 우리는 서버를 만든다.

MCP는 JSON-RPC 2.0 규약으로 메시지를 주고받습니다. “웹 API와 비슷하되, LLM 상호작용에 맞춰 설계된” 것.

→
도구 목록 요청 / 도구 호출 / 결과 반환
✓
다행히 FastMCP가 이 배선을 다 숨긴다 → 우린 함수만 작성

한 줄

JSON-RPC를 몰라도 된다. 데코레이터 단 함수가 곧 표준 메시지로 번역된다.

CH2세 기둥

MCP 서버가 노출하는 3가지

행동

Tools

POST 같은 호출 함수. 부작용 있음 — 메일 전송, DB 쓰기, 계산.

데이터

Resources

GET 같은 읽기 전용 데이터. URI로 주소 지정 — 문서·설정·로그.

템플릿

Prompts

재사용 프롬프트 템플릿. 호스트에서 슬래시 명령처럼 노출.

기억
오늘은 셋 다 만들어 본다. 내일(Day 4)은 그중 Resources를 깊게 설계한다.
/
CH2Tool vs Resource

헷갈릴 땐 — ‘부작용이 있는가’

Tool (행동/POST)

상태를 바꾸거나 외부에 영향. “티켓 생성”, “이메일 발송”, “주문”.

Resource (읽기/GET)

그냥 읽어오기. “정책 문서 읽기”, “설정값 조회”, “로그 보기”.

한 줄 규칙
부작용 있으면 Tool, 읽기 전용이면 Resource. 이 한 문장이 설계의 8할.
/
미니 실습분류

직접 해보기 — 무엇으로 만들까

과제 — 각 기능을 Tool / Resource로 분류

A
“오늘 환율 조회해 반환”
B
“회의록을 캘린더에 일정으로 등록”
C
“사내 휴가 정책 문서 읽기”
D
“두 수를 더한 결과 반환”

정답

C = Resource(읽기) / A·B·D = Tool(조회 호출·등록·계산처럼 ‘행위’). 외부 상태를 바꾸는 B는 분명한 Tool.

함수에 데코레이터만
FastMCP

왜tool·resource·prompt타입힌트→스키마
/
CH4왜 FastMCP

보일러플레이트를 지운다

원시 SDK로 직접 짜면 stdio 처리·JSON-RPC 직렬화·핸들러 등록이 번거롭습니다. FastMCP는 이를 데코레이터+타입힌트로 압축합니다.

✓
함수 시그니처 → JSON 스키마 자동 생성
✓
입력 검증·문서화·라우팅 자동
★
MCP 서버의 약 70%가 FastMCP 기반
install
# 설치 (현재 3.x)
pip install fastmcp

# 버전 확인
fastmcp version
/
CH4최소 예제

세 줄이면 유효한 MCP 서버

server.py
from fastmcp import FastMCP
mcp = FastMCP("Demo")

@mcp.tool
def add(a: int, b: int) -> int:
    """두 수를 더한다"""          # docstring → 모델이 읽는 설명
    return a + b

if __name__ == "__main__":
    mcp.run()                      # 기본 stdio 전송

핵심

타입힌트(int)와 docstring만으로 스키마·설명이 만들어진다. 3.x에선 데코레이터가 원함수를 그대로 돌려줘 직접 호출도 가능.

/
CH4나머지 둘

resource와 prompt도 같은 패턴

more.py
# 읽기 전용 데이터 — URI로 주소 지정
@mcp.resource("greeting://{name}", description="이름으로 인사말 반환")
def greeting(name: str) -> str:
    return f"안녕하세요, {name}님!"

# 재사용 프롬프트 템플릿 (슬래시 명령처럼)
@mcp.prompt
def summarize(topic: str) -> str:
    return f"다음 주제를 3문장으로 요약해줘: {topic}"

실무 팁

resource는 description=을 명시하라 — FastMCP의 docstring 파싱이 항상 잡히진 않는다(클라이언트 목록에서 설명이 비는 사고 방지).

재사용 프롬프트 템플릿

@mcp.prompt
def code_review(language: str, code: str) -> str:
    """코드 리뷰를 요청하는 프롬프트"""
    return f"다음 {language} 코드를 리뷰하고 개선점을 알려줘:\n\n{code}"

구분

Prompt는 ‘대화를 세팅’(템플릿), Tool은 ‘일을 실행’. Inspector의 Prompts 탭에서 인자를 넣어 결과 메시지를 확인.

LAB 5실전

쓸모 있는 도구 하나 — 메모 저장/검색

lab5_notes.py
NOTES: list[str] = []

@mcp.tool
def add_note(text: str) -> str:
    """메모를 추가한다(부작용 → Tool)"""
    NOTES.append(text); return f"저장됨 (총 {len(NOTES)}개)"

@mcp.tool
def search_notes(keyword: str) -> list[str]:
    """키워드가 든 메모를 찾는다"""
    return [n for n in NOTES if keyword in n]
✅ add_note로 몇 개 저장 → search_notes로 검색되는지 확인
