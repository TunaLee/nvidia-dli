/
CH1본질

Resource = GET 같은 ‘읽기 창구’

→
읽기 전용 — 상태를 바꾸지 않는다(부작용 없음)
→
URI로 주소 지정 — docs://policy/leave 처럼
→
모델이 필요할 때 읽어 컨텍스트에 넣는다

비유

도구(Tool)가 ‘버튼’이라면, 리소스(Resource)는 ‘서랍 속 문서’ — 열어서 읽을 수 있고, 건드려도 바뀌지 않는다.

/
CH1처음 짚기

URI vs URL — 리소스 ‘주소’가 뭔가

MCP에선
리소스 주소는 대개 URI(식별자) — config://status·docs://{id} 처럼 커스텀 스킴으로 지목만 한다. http로 못 열어도 정상 — 클라이언트가 read_resource(uri)로 받는다.
/
CH1두 종류

고정 리소스 vs 동적(템플릿) 리소스

static

고정 URI

config://status 처럼 하나의 고정 주소. 항상 같은 종류의 데이터를 반환.

dynamic

URI 템플릿

docs://{id} 처럼 매개변수를 받아 그에 맞는 데이터를 반환.

감각
템플릿은 ‘REST 경로 파라미터’와 같다 — {id}가 함수 인자로 들어온다.
/
CH1코드

DOCS = {"leave": "연차는 3영업일 전 신청", "expense": "비용은 영수증 첨부"}

@mcp.resource("docs://{doc_id}", description="사내 정책 문서를 ID로 읽는다")

CH3원칙

리소스를 잘 설계하는 법

일관된 URI 스킴

type://path/{param} 형태로 예측 가능하게. 종류별 접두사를 통일.

설명을 명시

description=로 무엇을 반환하는지 분명히 — 클라이언트 목록에서 보인다.

MIME 타입

텍스트·JSON 등 형식을 분명히 해 클라이언트가 잘 처리하게.

목록 제공

어떤 리소스가 있는지 ‘목록’으로 노출해 발견 가능하게.

/
CH3가장 흔한 실수

description= 빠뜨리면 생기는 일

✕
description 없음 — 목록에서 설명이 빈칸, 모델이 언제 쓸지 모름
✓
description 명시 — “무엇을·언제 읽나” 한 줄로 모델이 정확히 호출
description.py
# ✕ description 없음 → 목록에서 설명이 빈칸
@mcp.resource("docs://{id}")
def get_doc(id: str) -> str:
    return DOCS.get(id, "없음")

# ✓ description 명시 → 모델이 언제 읽을지 안다
@mcp.resource("docs://{id}",
              description="사내 정책 문서를 ID로 읽는다")
def get_doc(id: str) -> str:
    return DOCS.get(id, "없음")
규칙
리소스엔 항상 description=를 직접 적는다(FastMCP의 docstring 파싱은 신뢰 불가).
/
CH3발견·규모

목록으로 알리고, 크면 나눈다

→
클라이언트는 resources/list로 무엇이 있는지 발견한다
→
항목이 많으면 커서 기반 페이지네이션으로 나눠 전달
→
별도 인덱스 리소스도 두면 모델이 훑기 쉽다

왜 중요

목록이 없으면 모델은 무엇을 읽을 수 있는지 모른다. 발견 가능성이 곧 활용도.

CH4파일

폴더의 파일을 ID로 읽기

files.py
from pathlib import Path
BASE = Path("./docs")

@mcp.resource("docs://{name}", description="docs 폴더의 텍스트 파일을 읽는다")  # file://은 호스트로 해석돼 깨짐
def read_file(name: str) -> str:
    p = (BASE / name).resolve()
    if BASE.resolve() not in p.parents:   # 경로 탈출 방지(검증!)
        raise ValueError("허용되지 않은 경로")
    return p.read_text(encoding="utf-8")

CH7핵심 정리

핵심 정리 5가지

1

Resource = 읽기 전용·URI

static vs 템플릿

읽기=Resource, 행동=Tool
