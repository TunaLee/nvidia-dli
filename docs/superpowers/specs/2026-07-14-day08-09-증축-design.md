## 배경

Day07까지는 노트북에서 RAG·MCP·에이전트 루프를 **손으로** 만들었다. Day08부터는 **실제 코드 에이전트(Claude Code · Codex)를 켜고 진행**한다. 기존 덱·노트북은 이미 개념 + Claude Code 랩으로 상당 부분 구성돼 있으므로 **개념 슬라이드는 유지하고 랩을 재작성**한다.

## 관통 원칙

> **판단·도구·루프는 그대로, 하네스와 확장만 갈아입힌다.**
> **7강까지는 내가 만든다. 8강부터는 에이전트가 만들고 내가 판정한다.**

모든 실습은 3단으로 통일한다.

| 단계 | 내용 |
|---|---|
| **입힌다** | 무엇을 하네스에 얹나 (스킬·플러그인·훅·MCP·프레임워크) |
| **돌려본다** | 실제 Claude Code/Codex에서 실행 + 관찰 포인트 |
| **원리** | 무엇이 드러났나 (컨텍스트·권한·경계·오케스트레이션) |

## 1. Day08의 척추 — "하네스로 다시 만들기"

**핵심 발상**: 학생들은 이미 RAG(Day02)·MCP 서버(Day05)·에이전트 루프(Day07)를 손으로 짰다. 그걸 코드 에이전트에게 다시 만들게 한다.

**왜 강력한가**
- 학생에게 **정답(ground truth)이 있다.** 자기가 짜봤기 때문에 에이전트 출력을 **판정**할 수 있다. 에이전트 시대에 가장 어려운 능력이 바로 출력 판정이다.
- 한 바퀴가 닫힌다 — "내가 만든다" → "에이전트가 만들고 내가 판정한다".

**재구축 대상 (셋 다)**
1. **MCP 서버** (Day05/Day07 `day07-tools`) — 작고 명세가 분명해 첫 과제로 적합
2. **RAG 파이프라인** (Day02) — 임베딩 + 벡터검색. 큰 작업이라 컨텍스트 관리를 체감
3. **에이전트 루프** (Day07 `run_agent`) — 에이전트가 에이전트를 만든다

**킬러 시퀀스 — 자기가 만든 도구를 자기가 쓴다**

```
하네스에게 MCP 서버를 만들게 한다
  → claude mcp add 로 그 서버를 하네스에 등록한다
  → 하네스가 방금 자기가 만든 도구를 불러서 일한다
```

앞서 다룬 "도구가 없으면 만들어 붙인다"(메타)가 여기서 실물로 닫힌다.

## 3. Codex 병행 (인라인 대응 + 앵커)

Claude Code와 Codex는 **확장 원시요소가 다르다.** 스킬·플러그인·훅은 Claude Code에만 있다. **둘 다 있는 지점만** 나란히 대응시키고, 없는 지점은 "하네스마다 다르다"를 원리로 짚는다.

| 접점 | Claude Code | Codex |
|---|---|---|
| 설치·인증 | 네이티브 설치 | npm / standalone |
| 지침 파일 | `CLAUDE.md` | `AGENTS.md` |
| 권한·경계 | 권한 모드 (Shift+Tab) | 승인 모드 · 샌드박스 (`/permissions`) |
| MCP | `claude mcp add` | `codex mcp add` · `config.toml` |
| 헤드리스 | `claude -p` | `codex exec` |
| 메타 팩토리 | `revfactory/harness` | `meta-harness` (Codex 포트) — **미검증** |
| 멀티에이전트 프레임워크 | Superpowers · GSD | GSD (`--codex`) |
| 스킬·플러그인·훅 | 있음 | **없음** → 하네스별 차이를 원리로 |

**앵커**: Day08에 "Codex 대응표" 1장. 근거: `day08/Claude_Code_Codex_세팅.md` (작성 완료).

## 4. 실전 하네스 배치 (단일 ↔ 멀티 분리)

Superpowers·GSD는 **서브에이전트를 스폰하는 멀티에이전트**이므로 Day09로 보낸다.

| | **Day08 — 단일 에이전트** | **Day09 — 멀티에이전트** |
|---|---|---|
| 대상 | **GSTACK** (단일 세션 역할 파이프라인) + 토큰절감(**ponytail**, **caveman**) | **Superpowers** (TDD·서브에이전트) + **GSD** (스테이지별 신선 컨텍스트) |
| "이럴 땐 이걸" | 제품 관점 검토 → GSTACK / 토큰 절약 → ponytail·caveman | 품질 저하 → Superpowers / 긴 작업·문맥 저하 → GSD |
| Codex | — | GSD `--codex` |

- Day08의 GSD 슬라이드(현 45)는 **개념만 남기고 설치·실행은 Day09로 이관**.
- 마케팅 수치("92% 절감", "30K★")는 쓰지 않는다. caveman 각주(광고 65% vs 실측 ~8.5%)의 회의적 태도 유지.
- 슬42 "3대 프레임워크" 표와 슬43 "이 강의도 superpowers로 설계" 메타 노트는 재구성이 필요하다.

## 9. 미해결 — 강사 결정 필요 (강의 운영 블로커)

설계로는 못 푸는, **강의실에서 터질** 항목들.

1. **학생 계정** — Claude Code는 무료 플랜 불가(Pro/Max/Team/Enterprise/Console 필요). Codex도 유료. 학생 전원이 구독을 갖고 있나? 없으면 강사 시연 + 일부 실습으로 낮춰야 한다.
2. **토큰·레이트리밋** — Superpowers·GSD는 서브에이전트를 다수 스폰한다. Day09에서 Pro 플랜 학생은 수업 중 한도에 걸릴 수 있다.
3. **프레임워크 충돌** — GSTACK·Superpowers·GSD·harness를 한 저장소에 모두 설치하면 `.claude/` 네임스페이스(커맨드·에이전트·스킬)가 겹친다. **프로젝트 폴더 분리** 또는 worktree 필요.
4. **윈도우** — Claude Code는 네이티브 윈도우에서 샌드박싱 미지원(WSL2 필요). GSTACK·GSD·harness의 윈도우 동작은 미검증.
5. **`meta-harness`(Codex) 미검증** — Codex엔 `.claude/skills`가 없다. 실제로 무엇을 생성하는지 확인 필요.
6. **NVIDIA DLI 정체성** — Day08–09의 주역이 Claude/OpenAI 계정이다. 파이썬 시뮬만 NVIDIA Qwen을 쓴다. 강의 성격상 문제없는지 확인 필요.
