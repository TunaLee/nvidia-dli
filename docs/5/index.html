/
CH1본질

Tool = 버튼 — 누르면 작동한다

→
행동·부작용 — 상태를 바꾸거나 바깥과 상호작용
→
모델이 스스로 호출 — 필요할 때 판단해 부른다
→
입력·출력이 스키마로 명확히 정의된다

비유

Resource가 ‘서랍 속 문서’라면 Tool은 ‘버튼’ — 누르면 무언가 일어난다.

CH1힌트

도구에 성격을 표시한다

readOnly

읽기 전용

상태를 바꾸지 않음을 알린다.

destructive

파괴적

삭제·덮어쓰기처럼 위험함을 표시.

idempotent

CH2에러

모델이 스스로 고치게 만든다

✕ 불친절

“Error 500” — 모델이 무엇을 고쳐야 할지 모른다.

✓ 친절

“k는 1~50 사이여야 합니다” — 모델이 인자를 바로 고친다.

원칙
에러 메시지는 무엇이·왜 틀렸고·어떻게 고칠지를 담는다. 모델을 위한 안내문이다.

CH2안정성

→

→

→
불가능하면 중복 방지 키로 이중 실행을 막는다

예

CH3Context

도구가 바깥과 대화하는 창구

→

@mcp.tool
async def build(n: int, ctx) -> str:
    """n개를 처리하며 진행상황을 보고"""
    for i in range(n):
        await ctx.report_progress(i, n)
    return "완료"
/
CH3sampling

도구 안에서 모델에 되묻기

도구가 클라이언트의 LLM에게 생성을 요청할 수 있다(ctx.sample). 데이터를 가져와 요약까지 시켜 돌려준다.

→
서버 로직 + 모델 지능을 결합
→
방향이 서버→클라이언트로 흐른다
sample.py
@mcp.tool
async def summarize(url: str, ctx) -> str:
    """페이지를 가져와 모델에게 요약시킨다"""
    text = fetch(url)
    return await ctx.sample(f"3줄 요약:\n{text}")
/
CH3되묻기

실행 중 사용자에게 물어본다

→
도구가 부족한 정보를 실행 도중 사용자에게 요청(2025-06)
→
예: “어느 계정으로 보낼까요?”를 중간에 물어본다
→
위험한 작업 전 확인을 받는 데도 쓴다
감각
Tool은 한 번 부르고 끝이 아니라, 필요하면 사용자·모델과 주고받는다.
/
CH3비동기

기다리게 하지 말고 맡겨라

→
무거운 작업은 async로 논블로킹
→
진행상황을 알리며 응답성을 유지
→
분 단위 작업은 Tasks로 백그라운드 실행(2025-11)

왜

도구가 오래 멈춰 있으면 에이전트 전체가 멈춘다. 진척을 알리거나 맡기고 넘어간다.

@mcp.tool
async def bulk_add(items: list, ctx) -> str:
    """여러 할 일을 추가하며 진행상황 보고"""
    for i, it in enumerate(items):

CH8요약

한 장 치트시트

Tool이란

행동·부작용. 모델이 호출. 스키마로 계약.
